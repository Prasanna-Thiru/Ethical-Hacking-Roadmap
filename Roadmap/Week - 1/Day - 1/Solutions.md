# 1. What is Ethical Hacking?

Ethical Hacking is the authorized practice of bypassing system security to identify potential data breaches and threats in a network. The person who performs this is called an Ethical Hacker or White Hat Hacker — they do the same things a malicious hacker does, but with legal permission from the owner.

# 2. Types of Hackers?

White Hat — Ethical hackers who hack with permission and legal authorization /n
Black Hat — Malicious hackers who hack without permission for personal gain
Grey Hat — In between, no permission but no serious malicious intent
Script Kiddie — Uses existing tools without understanding how they work
Hacktivist — Hacks for political or social causes
State Sponsored — Government-backed hackers targeting other nations
Insider Threat — Someone within the organization who attacks from inside 

# 3. Cyber Kill Chain

A 7-step model developed by Lockheed Martin describing how a cyberattack unfolds:
Reconnaissance — Gathering information about the target
Weaponization — Creating malware or an exploit
Delivery — Sending the weapon via email, USB, or web
Exploitation — Triggering the vulnerability on the target
Installation — Installing a backdoor or malware
Command & Control (C2) — Remotely controlling the compromised system
Actions on Objectives — Stealing data or causing damage

# 4. Lab Setup Overview

Host Machine — Your main computer running Windows, Mac or Linux
Hypervisor — VirtualBox or VMware to run virtual machines
Kali Linux VM — Your primary attack machine
Target VM — Vulnerable machines like Metasploitable or DVWA
Network Setup — NAT or Host-Only network between VMs
Tools — Nmap, Burp Suite, Metasploit pre-installed in Kali.
